# 第一章.Filter过滤器

# 1.Filter过滤器介绍

1.概述:一个实现了特殊接口(Filter)的Java类. 实现对请求资源(jsp,servlet,html)的过滤的功能.  过滤器是一个"运行在服务器的程序", "优先于请求资源(Servlet或者jsp,html)之前执行" 过滤器是javaweb技术中"最为实用"的技术之一
    
2.特点:Filter是对客户端访问资源的过滤(资源都在服务器端),符合条件放行,可以正常访问web资源;不符合条件不放行,不能正常访问web资源,但是浏览器不知道为什么被拦截,但就是访问不了
  
3.作用:Filter的作用是对目标资源(Servlet,jsp)进行过滤，其应用场景有: 登录权限检查,解决网站乱码,过滤敏感字符等等

# 2.Filter过滤器入门1_1个Filter

1.创建一个Filter过滤器->Filter是一个接口,需要创建实现类去实现Filter接口
2.指明访问哪个资源时会经过过滤器
3.在过滤器中设置要不要放行的代码

再加一个过滤器,如果第一个过滤器不放行,就走不到第二个过滤器

@WebServlet("/servlet1")

public class MyFilter01 implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {

    }

    /*
      拦截方法

      如果该方法啥也不写,就会拦截

    @Override

          doFilter()放行方法
          注意:这里调用的doFilter不是递归
         */
        filterChain.doFilter(servletRequest,servletResponse);
    }

    @Override
    public void destroy() {

    }
}

# 5.处理中文乱码的Filter

<!DOCTYPE html>
<html lang="en">

<body>
<form action="/day14_Filter/servlet4" method = "post">
    <input type="text" name="username">
    <input type="submit" value="提交">

@WebServlet("/servlet4")
public class MyServlet04 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        String username = request.getParameter("username");
        System.out.println(username);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

public class MyFilter04 implements Filter {
    public void destroy() {
    }

    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) throws ServletException, IOException {
        //处理请求参数中文乱码问题
        req.setCharacterEncoding("utf-8");
        chain.doFilter(req, resp);
        System.out.println("我是过滤器4");
    }

    public void init(FilterConfig config) throws ServletException {

    }
}


    <!--处理中文乱码的过滤器-->
    <filter>
        <filter-name>filter4</filter-name>
        <filter-class>com.atguigu.b_filter.MyFilter04</filter-class>
    </filter>
    <filter-mapping>
        <filter-name>filter4</filter-name>
        <url-pattern>/servlet4</url-pattern>
    </filter-mapping>

注意:

处理请求乱码可以写在过滤器中

但是处理响应乱码放到过滤器中不合适

原因:如果我们处理的不是一个文本,而是一个json数据,或者是其他的文件类型

​ 我们写response.setContentType("text/html;charset=utf-8");就不合适了,此方式只是处理文本,不能处理其他类型

# 6.Filter过滤器生命周期以及生命周期方法

生命周期阶段执行时机生命周期方法
创建对象Web应用启动时init()方法，通常在该方法中做初始化工作
拦截请求在每次过滤请求的时候执行doFilter()方法，通常在该方法中执行拦截过滤
销毁过滤器销毁前执行destroy()方法，通常在该方法中执行资源释放

# 7.Filter过滤器4种匹配规则

# 7.1.精准匹配

访问哪个资源,被过滤器拦截,那么过滤器中就写哪个资源的路径
比如:要过滤Servlet6,那么就写Servlet6的url-pattern映射路径

    <filter>
        <filter-name>filter6</filter-name>
        <filter-class>com.atguigu.b_filter.MyFilter06</filter-class>
    </filter>
    <filter-mapping>
        <filter-name>filter6</filter-name>
        <url-pattern>/servlet6</url-pattern>
    </filter-mapping>

# 8.Filter过滤器链以及执行顺序

1.什么叫过滤链:
  一个请求可能被多个过滤器所过滤，只有当所有过滤器都放行，请求才能到达目标资源，如果有某一个过滤器没有放行，那么请求则无法到达后续过滤器以及目标资源，多个过滤器组成的链路就是过滤器链
      
2.执行顺序:
  a.如果过滤器的匹配路径都是用的<url-pattern>那么哪个过滤器在web.xml中前面,哪个过滤器先执行
  b.如果其中一个过滤器用的是<servlet-name>,一个过滤器用的<url-pattern>过滤请求,无论使用<servlet-name>的过滤器先配置还是后配置,都会先走<url-pattern>过滤请求的过滤器

# 第二章.Listener监听器(了解)

# 1.Listener监听器介绍

1.监听器：专门用于对其他对象身上发生的事件或状态改变进行监听和相应处理的对象，当被监视的对象发生情况时，立即采取相应的行动。

2.Servlet监听器:Servlet规范中定义的一种特殊类，它用于监听Web应用程序中的ServletContext，HttpSession 和HttpServletRequest等域对象的创建与销毁事件，以及监听这些域对象中的属性发生修改的事件。
    
3.监听器作用:主要是监听域对象的,比如监听域对象的创建,销毁,以及属性的改变等
    
4.过滤器和监听器区别:
  过滤器:过滤的是请求
  监听器:监听的是域对象的状态

# 2.监听器分类(了解即可)

# 2.1.ServletContextListener

作用：监听ServletContext对象的创建与销毁
方法名作用
contextInitialized(ServletContextEvent sce)ServletContext创建时调用
contextDestroyed(ServletContextEvent sce)ServletContext销毁时调用
ServletContextEvent对象代表从ServletContext对象身上捕获到的事件，通过这个事件对象我们可以获取到ServletContext对象。

# 2.2.HttpSessionListener

作用：监听HttpSession对象的创建与销毁

# 3.ServletContextListener的使用

1.介绍:ServletContextListener是监听ServletContext对象的创建和销毁的，因为ServletContext对象是在服务器启动的时候创建、在服务器关闭的时候销毁，所以ServletContextListener也可以监听服务器的启动和关闭
    
2.使用场景:
  将来学习SpringMVC的时候，会用到一个ContextLoaderListener，这个监听器就实现了ServletContextListener接口，表示对ServletContext对象本身的生命周期进行监控。

# 3.1.代码演示

编写监听器的步骤:
1. 写一个类实现对应的：Listener的接口(我们这里使用的是ServletContextListener),并且实现它里面的方法
   1.1 contextInitialized()这个方法在ServletContext对象被创建出来的时候执行，也就是说在服务器启动的时候执行
   1.2 contextDestroyed()这个方法会在ServletContext对象被销毁的时候执行，也就是说在服务器关闭的时候执行
   
2.在web.xml中注册(配置)监听器

public class ContextLoaderListener implements ServletContextListener {

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("在服务器启动的时候，模拟创建SpringMVC的核心容器...");
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("在服务器启动的时候，模拟销毁SpringMVC的核心容器...");
    }
}

修改LoginFilter类:
1.先从session域中取出用户对象
2.如果用户对象不是空,证明用户登录了,就放行
3.如果用户对象为空,证明用户尚未登录
  A:同步请求,转发到登录页面
  B:异步请求,响应尚未登录的信息
=================================================================
@WebFilter("/protected/*")
public class LoginFilter implements Filter {
    public void destroy() {
    }

    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) throws ServletException, IOException {
        //1.由于参数都是父接口,所以先强转一下
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) resp;
        //2.获取session中的user对象
        User user = (User) request.getSession().getAttribute("user");
        //3.如果user为空,开始拦截
        if (user == null) {
          /*
            注意:
              我们点击"购物车"的时候是同步请求
              我们点击"添加购物车"以及其他购物车操作都是异步的

              我们如何判定是同步还是异步呢?
            解决:
              a.我们点击"购物车"按钮其实在index.html中调用了method=toCartPage方法
                如果我们获取到的method请求参数是toCartPage就是同步请求

              b.其他的购物车操作都是异步的
           */
          //4.获取method请求参数
            String method = request.getParameter("method");
            if ("toCartPage".equals(method)){
                response.sendRedirect(request.getContextPath()+"/user?method=toLoginPage");
            }else{
                //异步请求
                CommonResult commonResult = CommonResult.error().setMessage("unlogin");
                //打成json数据,响应给客户端
                JsonUtils.writeResult(response,commonResult);
            }
        } else {
            chain.doFilter(req, resp);
        }

    }

    public void init(FilterConfig config) throws ServletException {

    }

}

# 第四章.ThreadLocal

# 1.ThreadLocal介绍

1.ThreadLocal概述:是一个容器
2.作用:存放一个变量或者一个对象
3.方法:
  void set(存放的数据) -> 往ThreadLocal中存储数据
  get() -> 获取数据
4.特点:
  a.一次只能存储一个元素,获取的也是最后一次存进去的元素
  b.底层实现原理为Map集合
    key指的是当前线程
    value存的是我们想要存的值
  c.当我们的值保存到ThreadLocal中,那么这个值就会跟当前线程绑定
5.注意:
  一个线程往ThreadLocal中存储数据,其他线程获取不到

public class Test01 {
    public static void main(String[] args) {
        //创建ThreadLocal对象
        ThreadLocal<String> local = new ThreadLocal<>();
        //local.set("多尔衮");
        local.set("涛哥");
        System.out.println(local.get());
        
        new Thread(()-> System.out.println(local.get())).start();
    }
}
