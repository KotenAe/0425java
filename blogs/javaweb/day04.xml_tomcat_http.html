# 4.xml的概述

1.XML ：可扩展标记语言（EXtensible Markup Language）
        XML 它是一种标记语言，很类似 HTML，xml标签都是自定义的。 如：<user></user> 或 <student></student>
        
2.W3C在1998年2月发布1.0版本，2004年2月又发布1.1版本，但是因为1.1版本不能向下兼容1.0版本，所以1.1没有人用。同时，在2004年2月W3C又发布了1.0版本的第三版。我们要学习的还是1.0版本。

# 5.xml和html的差异

1.xml标签都是自定义的，html标签是预定义。
2.xml的语法严格，html语法松散。
3.xml是存储数据的，html是展示数据。

由标签组成,标签名可以随便写都是自定义的,但是开发中毕竟我们是一个团队,不能大家各写一套xml配置文件,所以xml是有约束的

# 10.xml中的xml文档约束

1.问题:
  xml中的标签名可以自定义,但是我们将来开发是一个team,所以我们的项目中的xml文档,就不能你写你的,我写我的,所以这样将来项目整合时会出现大问题,我们应该统一一下标签名
      
2.所以,将来使用xml的时候是有文档约束的

3.文档约束作用:约束我们的xml中的标签名以及应该写啥属性
    
4.xml的文档注释:
  dtd约束
  schema约束
5.注意:
  xml约束将来不是我们自己写的,都是框架提前写好的,用什么框架,就必须按照人家框架规定好的约束写xml

# 11.根据DTD约束编写xml文档

直接粘贴dtd文件,粘贴一以下代码:

<?xml version="1.0" encoding="UTF-8"?>
<!--
	尚硅谷DTD教学实例文档。
	模拟spring规范，如果开发人员需要在xml使用当前DTD约束，必须包括DOCTYPE。
	格式如下：
	<!DOCTYPE beans SYSTEM "bean.dtd">
-->
<!ELEMENT beans (bean*,import*) >
<!ELEMENT bean (property*)>
<!ELEMENT property (#PCDATA)>

<!ELEMENT import (#PCDATA)>

<!ATTLIST bean id CDATA #REQUIRED
			   className CDATA #REQUIRED
			   type CDATA #IMPLIED
>

<!ATTLIST property name CDATA #REQUIRED
			   	   value CDATA #REQUIRED
>
<!ATTLIST import resource CDATA #REQUIRED>

<?xml version="1.0" encoding = "UTF-8" ?>
<!--
  beans代表的是根标签
  bean.dtd:表示的是约束文档在本地上的路径
-->
<!DOCTYPE beans SYSTEM "bean.dtd">
<beans>
    <bean id="xa001" className="java">
        <property name="tom" value="sgg"></property>
    </bean>
    <import resource="path"></import>
</beans>

注意:

想要让我们自己的xml被dtd约束文档约束,必须让这两个文档产生联系:

<!DOCTYPE beans SYSTEM "bean.dtd">

# 12.DTD约束阅读

各部分解释:
  DOCTYPE:文档类型
  beans:根标签
  SYSTEM:本地系统文件
  "bean.dtd":产生关系的文件名,写文件的路径
  bean*:可以写多个bean标签
  (bean*,import*):,表示标签顺序
  #PCDATA:此标签体中应该是文本
  CDATA:代表的是标签中的属性值为字符型
  #REQUIRED：属性值必须设置

		1. <element> 声明元素(标签)

					</complexType>
				</element>
				<element name="import">
					<complexType>
						<attribute name="resource" use="required"></attribute>
					</complexType>
				</element>
			</choice>
		</complexType>
	</element>

# 16.dom4j解析器的核心API介绍

1.解析方式:
  a.DOM:常用->将整个xml文档加载到内存中,生成一个对应的document对象(代表的是xml文档),会生成一个dom树
    缺点:如果xml文档太大,有可能会造成内存溢出 
    优点:完整的保留xml的结构,好操作
    
  b.SAX:速度快,更有效的解析方式,逐行解析,将一行加载到内存,解析,解析完毕,释放
  c.PULL:类似于SAX解析,主要用于安卓
  
2.解析器:根据不同的解析方式创造出的不同的api

3.DOM解析思想:剥洋葱
  a.先获取根标签
  b.根据根标签获取下面的子标签
  c.根据获取出来的标签,就可以去获取该标签中对应的属性值,文本部分(标签体)


1.核心类:SaxReader(不是Sax解析的核心类)
        方法:Document read(绑定这个xml文件的输入流)->读取xml
2.Document对象
        方法:Element getRootElement()->获取根标签
3.Element对象:
        方法:
            a.List<Element> elements() -> 获取根标签下的所有子标签(不包含孙子标签)
            b.String attributeValue(String 属性名)-> 根据属性名获取对应的属性值
            c.String getText()->获取的是标签体的文本部分

# 17.利用dom4j解析xml文档代码实现

使用dom4j解析需要导jar包:dom4j-1.6.1.jar

<?xml version="1.0" encoding="UTF-8" ?>
<beans>
    <bean id = "001" className = "cn.atguigu.demo.User">
        <property name = "user" value = "jack">杰克</property>
        <property name = "user" value = "rose">肉丝</property>
    </bean>

    <bean id = "002" className = "cn.atguigu.demo.Admin">
        <property name = "user" value = "admin">管理员</property>
        <property name = "user" value = "write">怀特</property>
    </bean>
</beans>

注意:将来所有的配置文件都要放到resources资源文件夹中

1.在当前模块下创建一个文件夹:取名为-> resources
2.将普通文件夹变成资源文件夹->对着文件夹右键-> mark directory as->resource root

public class Test01_dom4j {
    public static void main(String[] args) throws Exception {
        //1.创建SaxReader对象
        SAXReader saxReader = new SAXReader();
        //2.创建字节输入流(InputStream),读取配置文件
        InputStream in = Test01_dom4j.class.getClassLoader().getResourceAsStream("beans.xml");
        //3.利用SaxReader中的read方法,将流中的文件数据加载到内存中,返回Document对象
        Document document = saxReader.read(in);
        //4.获取根标签:getRootElement(),返回Element对象
        Element rootElement = document.getRootElement();
        //5.调用Element对象中的elements()获取根标签下的子标签,返回一个List集合,泛型为Element
        List<Element> list = rootElement.elements();
        //6.遍历List集合,将每一个子标签对象获取出来
        for (Element beanElement : list) {
        //7.获取子标签中的属性值和标签体的文本部分: attributeValue("属性名")  getText()
            String id = beanElement.attributeValue("id");
            String className = beanElement.attributeValue("className");
            System.out.println(id+"..."+className);

            //8.还可以获取下一级子标签
            List<Element> propertyList = beanElement.elements();
            for (Element propertyElement : propertyList) {
                String name = propertyElement.attributeValue("name");
                String value = propertyElement.attributeValue("value");
                String text = propertyElement.getText();

                System.out.println(name+"..."+value+"..."+text);
            }

        }
    }
}

# 18.XPath解析XML

XPath 是一门在 XML、html 文档中查找信息的语言。

XPath 是一个 W3C 标准，可通过W3CSchool文档查阅语法

由于DOM4J在解析XML时只能一层一层解析，所以当XML文件层数过多时使用会很不方便，结合XPATH就可以直接获取到某个元素

xpath解析作用:支持查询表达式,快速定位到想要的标签位置

# 18.1使用dom4j支持xpath具体操作

默认的情况下，dom4j不支持xpath，如果想要在dom4j里面使用xpath，需要引入支持xpath的jar包 jaxen-1.1.6.jar

所以,我们想使用xpath解析,就需要先导入xpath的jar包,然后在使用

注意:xpath需要和dom4j配合使用

在dom4j里面提供了两个方法，用来支持xpath(Element中的方法)

Node是Element的父类
List<Node> selectNodes("xpath表达式")  -> 用来获取多个标签
           selectNodes("//BBB")->  获取名字为BBB的标签
    
Node selectSingleNode("xpath表达式"),用来获取第一个标签对象
     selectSingleNode("//BBB")->获取第一个BBB标签对象

# 18.2 xpath表达式常用查询形式

  • 第一种查询形式

    /A/B/C/   表示一层一层的,获取的是a下面的b下面的c标签

  • 第二种查询形式(最常用)

    //BBB  表示只要是标签名为BBB都能得到

  • 第三种查询形式

    /*    所有标签

  • 第四种查询形式(没啥用)

    BBB[1]:表示第一个BBB元素
BBB[last()]: 表示最后一个BBB元素

  • 第五种查询形式

    //BBB[@id]:获取的是有id属性的BBB标签

  • 第六种查询形式

    //BBB(@id = 'b1')  表示元素名称为BBB,在BBB上面有id属性,并且id的属性值是b1

        //4.获取根标签:getRootElement(),返回Element对象
        Element rootElement = document.getRootElement();
        //5.获取所有的age
        List<Element> list = rootElement.selectNodes("/students/student/age");
        for (Element element : list) {
            System.out.println(element.getText());
        }

        System.out.println("===================================");
        //6.获取标签sex
        List<Element> list1 = rootElement.selectNodes("//sex");
        for (Element element : list1) {
            System.out.println(element.getText());
        }

        System.out.println("===================================");

        /*
         7.获取第一个sex

           注意:Node类是Element类的父类
         */
        //Node node = rootElement.selectSingleNode("//sex");
        Element element = (Element) rootElement.selectSingleNode("//sex");
        System.out.println(element.getText());

    }
}

# 第二章.Tomcat服务器软件

# 1.Tomcat介绍

Tomcat服务器软件是一个开源的轻量级的web应用服务器,在中小型系统和并发量小的场合下被普遍使用,是开发和调试Servlet,jsp程序的首选,也是绝大多数JavaEE开发工程师所必用的
    
我们开发的web项目需要部署到tomcat下,启动tomcat,我们就可以在浏览器上访问项目下的web资源了

# 2.Tomcat安装

1.tomcat安装很简单->直接解压即可
2.注意:
  1.解压tomcat的路径,不要有中文,不要有空格,不要有乱七八糟的符号(只要是java相关的软件都这样)
  2.tomcat是java写的,所以运行tomcat,计算机上必须要有正确的java环境变量

# 3.Tomcat目录结构

1.bin: 脚本目录(重点)
       启动tomcat脚本:startup.bat
       停止tomcat脚本:shutdown.bat
2.conf:配置文件目录
       核心配置文件:server.xml
       用户权限配置文件:tomcat-users.xml
       所有web项目默认配置文件:web.xml
3.lib:依赖库,tomcat和web项目中需要使用的jar包
 
4.logs:日志文件目录
    
5.temp:临时文件目录,内容可以随便删除(不用管)
    
6.webapps:默认情况下,发布的web项目所存放的目录(重点)
    
7.work:tomcat处理jsp页面的工作目录

# 4.Tomcat端口简介_8080

在tomcat/conf/server.xml,这个文件中配置了tomcat的端口号

8080:tomcat启动之后,用于建立http连接(重中之重)
8005:tomcat断开停止之后所用端口
8009:与其他http服务器通信接口,采用二进制传输可读性文本

以上三个端口启动tomcat之后,都会开启,所以一旦开启之后三个端口都会被占用

# 5.Tomcat的访问

1.启动tomcat:进入bin路径下,找到startup.bat脚本,双击
2.停止tomcat:进入bin路径下,找到shutdown.bat脚本,双击 或者 直接将黑窗口x掉
    
3.访问:在浏览器地址栏上输入:  主机地址:8080

# 6.Tomcat下部署web项目_手动

1.我们需要将web项目打包成war包,将war包放到tomcat的webapps下(后面会学)
2.启动tomcat之后,tomcat会自动将war包解压

# 7.Tomcat直接访问Web项目名(conf/web.xml)

2.原因:

# 第三章.HTTP协议

# 1.HTTP协议介绍

1.HTTP协议:超文本传输协议(HyperText Transfer Protocol)是互联网上应用最为广泛的一种网络协议.用于定义web浏览器与web服务器之间交换数据的过程
    
2.web浏览器和web服务器数据交换过程:
  web浏览器向web服务器发送请求(携带请求数据)
  web服务器向web浏览器返回响应(携带响应数据)
      
3.http协议默认端口:80

# 2.HTTP两种报文格式

1.请求报文(客户端往服务端发送请求时的一些请求数据内容)
2.响应报文(服务端往客户端响应结果时的一些响应数据内容)

# 3.HTTP请求报文

index.html
============================================================
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>首页</title>
</head>
<body>
<form action="login.html" method = "get">
    用户名:<input type = "text" name = "username"/><br>
    密&nbsp;&nbsp;&nbsp;码:<input type = "password" name = "password"/><br>
    <input type="submit" value = "登录">
</form>
</body>
</html>

login.html
============================================
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
登录成功
</body>
</html>

  a.get请求没有请求体(请求体中都是放请求参数的)
