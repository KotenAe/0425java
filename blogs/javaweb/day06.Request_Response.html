# 1.HttpServletRequest概述

1.概述:
  在Servlet中定义了一个类ServletRequest,ServletRequest有一个子类HttpServletRequest(请求对象)
      
2.作用:
  HttpServletRequest对象封装了客户端发到服务器端上的请求信息
  a.一个请求信息:请求行,请求头,请求体(请求参数)
  b.想要拿到请求行,请求头,请求体中的数据,就需要从HttpServletRequest对象中获取 
      
3.我们需要掌握的就是获取请求体中的数据,因为请求体中有请求参数

# 1.1.HttpServletRequest的组成

1.请求行:
  get请求:请求方式  请求路径?请求参数  协议和版本
  post请求:请求方式  请求路径  协议和版本
2.请求头:
  对服务器的解析提供一些指导性信息
3.请求体:
  get请求:没有请求体,请求参数都在地址栏上
  post请求:是有请求体的,请求参数在请求体中

# 1.2.HttpServeltRequest的作用

1.获取HTTP请求三部分中的数据(行,头,体),主要是获取请求体中的请求参数
2.进行请求转发
3.作用域对象对数据进行存取

# 2.1.获取请求行方法:

方法名说明
getMethod()获取请求方式(会用)
getContextPath()获取当前应用上下文路径(在application context中设置的名字)(会用)
getRequestURI()获取请求地址,不带主机名
getRequestURL()获取请求地址,带主机名
@WebServlet("/request1")
public class Request1 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //getMethod() 获取请求方式: get  post
        String method = request.getMethod();
        System.out.println("method = " + method);
        System.out.println("=======================");

        //getContextPath() 获取web应用名称
        String contextPath = request.getContextPath();
        System.out.println("contextPath = " + contextPath);
        System.out.println("=======================");
        //getRequestURI()获取的是    web应用名称/具体访问资源  不带主机名
        String requestURI = request.getRequestURI();
        System.out.println("requestURI = " + requestURI);
        System.out.println("=======================");
        /*
          getRequestURL() 获取请求地址,带主机名
          http://localhost:8080/day06_request_response/request1

          作用:将来开发是分布式,我们需要部署多个tomcat,多个tomcat之间的端口号还不能一样
              所以用户访问的是哪个tomcat下的模块,我们作为程序员是需要知道的
              所以此方法就有用了
         */
        StringBuffer requestURL = request.getRequestURL();
        System.out.println("requestURL = " + requestURL);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 2.3.获取请求体中的请求参数(重点)

1.什么时候请求参数:
  a.客户端发送给服务器端的请求数据
  b.get请求的请求参数: web应用名称?请求参数
  c.post请求的请求参数:在请求体中
    不管是post,get,最终都会将请求参数和请求路径拼接在一起
2.请求参数分为3种:
  a.表单提交之后?后面的请求参数->get
  b.表单提交之后,在请求体中 ->post
  c.ajax发送异步请求携带的请求参数(后面再学)

3.请求参数,不管是get,post
  key=value&key=value
  
  key代表的是页面上的name属性值
  value代表的是页面上value的属性值
      
  所以我们获取请求参数都是根据页面上name的属性值获取value的属性值
方法名返回值类型方法描述
request.getParameter("请求参数的name属性名")String根据提交参数的name获取对应的value
request.getParameterValues("请求参数的name属性名")String[]根据提交参数的name获取多个value
request.getParameterMap()Map<String,String[]>获取提交请求的所有参数,以键值对的方式 存到map中

@WebServlet("/request3")
public class Request3 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
       /*
         request.getParameter("请求参数的name属性名")

         根据name获取value
        */

        String username = request.getParameter("username");
        String password = request.getParameter("password");
        String nickname = request.getParameter("nickname");
        String email = request.getParameter("email");
        System.out.println(username+"..."+password+"..."+nickname+"..."+email);

        System.out.println("==================================");

        /*
          request.getParameterValues("请求参数的name属性名")
          返回值:String[]

          根据提交参数的name获取多个value

          主要是根据页面上相同的name获取对应的多个value
         */

        String[] hobbyArray = request.getParameterValues("hobby");
        System.out.println(Arrays.toString(hobbyArray));

        System.out.println("==================================");

        /*
          request.getParameterMap()

          Map<String,String[]>

          获取提交请求的所有参数,以键值对的方式 存到map中
          
          map中的value数组代表的是请求参数中的value值

         */

        Map<String, String[]> map = request.getParameterMap();
        Set<Map.Entry<String, String[]>> entrySet = map.entrySet();
        for (Map.Entry<String, String[]> entry : entrySet) {
            String[] value = entry.getValue();
            //遍历数组,将请求参数中的value值获取出来
            for (String s : value) {
                System.out.println(s);
            }
        }
        
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 2.4.解决获取请求参数乱码

# 2.4.1.为什么会发生请求参数乱码问题

1.为什么:客户端往服务器端发送请求,会将请求参数进行一个编码
        服务器接收到请求,解析请求会进行解码
        在这个过程中,如果两端编码和解码遵守的编码规则不一致,就会乱码

2.注意:
  浏览器发送给服务器端数据时用的是UTF-8编码
  tomcat采用的编码格式:ISO8859-1
      
3.在服务端需要告诉tomcat采用什么编码去解码

# 2.4.2.怎么解决请求参数乱码问题(主要针对post请求)

1.注意:
  get请求如果请求参数中有中文,不用指定编码去解
  post请求如果请求参数中有中文,需要在服务器端指定编码
2.为啥?
  tomcat8针对于get请求已经做了优化了,所以发送get请求不用处理中文乱码
  post请求没有被tomcat8优化,所以如果当发送post请求的时候,我们自己需要在服务器端指定编码
方法名说明
request.setCharacterEncoding("UTF-8")将获取到的请求参数变成utf-8格式

@WebServlet("/request4")
public class Request4 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        /*
          request.setCharacterEncoding("UTF-8")
          将获取到的请求参数变成utf-8格式
         */
        request.setCharacterEncoding("UTF-8");


        //获取请求参数
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        System.out.println("username = " + username);
        System.out.println("password = " + password);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 3.请求转发

# 3.1.什么是请求转发

1.概述:通过项目下的一个资源将请求转发到另外一个项目资源下
2.注意:
  在请求转发的过程中,我们只会产生一次请求,而且地址栏不会改变
  因为请求转发是服务器内部动作
  浏览器不知道服务器内部干了啥,浏览器只知道能展示页面就可以了,至于怎么展示,浏览器不知晓

# 3.2.请求转发方法

方法名说明
RequestDispacher getRequestDispacher(String path)获取请求转发器对象
参数:path指的是要转到哪个servlet资源上去或者哪个页面上去
1.如果要转发到另一个servlet上,就写servlet对应的url-pattern
2.如果要转发到某一个页面上,就写这个页面路径
forward(request对象,response对象)实现请求转发方法
此方法是RequestDispacher对象的方法

# 3.3.请求转发演示

需求:浏览器访问request5,然后request5将请求转发到request6,最终响应由request6响应

          返回值:RequestDispacher对象

          需要再次调用RequestDispacher对象中的forward(request,response)真正实现请求转发

        //request.getRequestDispatcher("/request6").forward(request,response);

# 3.4.总结

1.请求转发的跳转是服务器内部行为,在这个过程中浏览器只会发起一次请求
2.请求转发只能跳转到本项目的资源中去,包括WEB-INF下的资源
3.请求转发不会改变地址栏的地址
4.请求转发这一套动作都是在一个请求中

# 第三章.BeanUtils工具类的使用

1.问题:
  a.我们将整个表单提交给服务器,我们为了方便,调用了getParameterMap()可以一次性将请求参数获取到
  b.但是getParameterMap()返回的是Map集合,key是页面上的name属性值,value是String数组,代表value
  c.将来我们会把String数组中的请求参数数据放到sql语句中,我们需要将String数组传递到dao层
    遍历String数组,挨个将String数组中的请求参数一个一个的放到sql语句的?占位符上
    String sql = "select * from user where username = ? and password = ?"
    遍历String数组,然后根据索引获取元素,放到对应的?上,这样做很麻烦
      
2.解决:
  如果我们直接将Map中的value一次性封装到一个JavaBean对象中,然后将Javabean对象传递到dao层
  那么我们给?赋值的时候,只需要调用JavaBean中的get方法就能赋值了
   
3.工具类:BeanUtils
  作用:可以将Map中的数据一次性封装到JavaBean中

BeanUtils:使用
1.导jar包:
2.创建一个JavaBean类,属性要和表单上提交过来的name属性值保持一致
方法名说明
BeanUtils.populate(Javabean对象,Map集合)将Map集合中的数据直接封装到Javabean中

    }
}

# 第四章.HttpServletResponse响应对象

# 1.HttpServletResponse响应对象介绍

# 1.1.HttpServletResponse概念

1.概述:
  在Servlet中定义了一个类ServletResponse,ServletResponse有一个子类HttpServletResponse(响应对象)

# 1.2.HttpServletResponse作用

1.HttpServletResponse对象可以对浏览器设置一些响应信息,所以可以用里面的方法去设置响应信息参数
2.响应信息:
  响应行: 协议版本号  响应状态码
  响应头: 对浏览器解析的一些指导性信息
  响应体: 浏览器要展示的部分(重点)

# 2.HttpServletResponse使用

# 2.1.设置响应状态码(了解即可)

1.tomcat会根据不同的情况去自动设置响应状态码
2.常见的响应状态码:
  200:响应成功
  302:重定向
  404:找不到资源
  500:服务器内部错误
方法名说明
response.setStatus(状态码)设置响应行中的响应状态码

# 2.3.3.方法三.使用字节流做响应

字符流:针对文本文档
字节流:万能流,使用字节流可以响应文本,图片等
      使用字节流做响应,响应中文不乱码
方法名说明
ServletOutputStream getOutputStream()获取字节流对象
@WebServlet("/response5")
public class Response5 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //1.获取输出流,将读到的文件字节,响应给浏览器
        ServletOutputStream os = response.getOutputStream();

        ServletContext context = getServletContext();
        String realPath = context.getRealPath("1.jpg");

        //2.读取本地上的图片
        FileInputStream in = new FileInputStream(realPath);
        //3.边读边写
        int len;
        while((len = in.read())!=-1){
            os.write(len);
        }

        //关流-> response中的流不用我们自己关
        in.close();
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 2.4.设置浏览器重定向

1.概述:
  从一个web项目资源上跳转到另外一个资源上,在这个过程中,会发起两次请求
方法名说明
response.sendRedirect("重定向的资源路径")请求重定向到另外一个资源中去
1.需求:访问response6,重定向到response7中,让response7作响应

@WebServlet("/response6")
public class Response6 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        request.setAttribute("username","root");

        //重定向到response7上
        response.sendRedirect(request.getContextPath()+"/response7");
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

@WebServlet("/response7")
public class Response7 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.setContentType("text/html;charset=utf-8");
        response.getWriter().write("我是response7");

        /*
          从请求域中获取的是null
          因为重定向是两次请求

          而请求域中的数据只能在同一个请求中获取
         */
        Object username = request.getAttribute("username");
        System.out.println("username = " + username);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 2.5.重定向和请求转发区别

1. 重定向会由浏览器发起新的请求，而请求转发不会发起新的请求
2. 重定向可以访问任意互联网资源，而请求转发只能访问本项目资源
3. 重定向不能访问本项目的WEB-INF内的资源，而请求转发可以访问本项目的WEB-INF内的资源
4. 发起重定向的资源和跳转到的目标资源没在同一次请求中，所以重定向不能在请求域中使用；而发起请求转发的资源和跳转到的目标资源在同一次请求中，所以请求转发可以在请求域中使用
