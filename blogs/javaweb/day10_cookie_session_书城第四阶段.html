# 第一章.会话介绍

# 1.会话技术介绍

1.保持用户登录状态，就是当用户在登录之后，会在服务器中保存该用户的登录状态，当该用户后续访问该项目中的其它动态资源(Servlet或者Thymeleaf)的时候，能够判断当前是否是已经登录过的。而从用户登录到用户退出登录这个过程中所发生的所有请求，其实都是在一次会话范围之内
    
2.其实我们当打开浏览器访问百度,只要是连接上了百度服务器,浏览器不关,就保持在一次会话之内,这一次会话中可以包含多个请求,多个响应
  如果客户端和服务端断开,一次会话结束,再次开启,下一次会话开启了

# 2.域对象范围

# 2.1.应用域范围_ServletContext

1.ServletContext叫做全局域,代表整个web项目,所以在此域中存储的数据,只要是不离开当前web项目随时可以拿到

# 2.2.请求域范围_Request

1.request叫做请求域:当一个请求往请求域中存储了一个数据,那么就只能在同一个请求中,从域中获取数据
2.脱离了此请求,那么在其他请求中就不能拿到第一个请求域中的数据了

# 2.3.会话域范围_Session

1.session域:会话域
2.一次会话什么时候开始:当浏览器连接到服务器的那一刻开始,会话就开始了
  在这一次会话中,我们可以往服务器发送多次请求,也能收到多次响应
3.所以,在会话域中存储的数据,只要会话不断开,在此会话中的多次请求中都可以拿到会话域中的数据
4.一次会话什么时候结束:服务器和浏览器断开

# 第二章.浏览器会话技术_Cookie

# 1.Cookie介绍

1.概述:
  Cookie是浏览器端的一个技术,这个Cookie对象是由服务器端创建的,保存在浏览器端的一个小数据对象,从这里面存储的key和value都是String的
 
2.作用:
  在浏览器端存储数据
      
3.使用场景:
  a.记录一下用户信息
  b.记录视频播放进度
      
4.Cookie一定要记住:
  a.从服务器端产生的
  b.保存到浏览器上
  c.下一次访问服务器,会携带cookie,发送到服务器端

# 2.Cookie入门

方法说明
Cookie(String key,String value)创建cookie对象,存放数据
response.addCookie(cookie对象)将cookie对象响应给浏览器
request.getCookies()获取cookie对象,返回一个cookie数组
getName()获取cookie的key
getValue()获取cookie中的value
@WebServlet("/cookie1")
public class Cookie1 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //1.创建Cookie对象
        Cookie cookie = new Cookie("username", "aobama");
        //2.将cookie响应给浏览器
        response.addCookie(cookie);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

@WebServlet("/cookie2")
public class Cookie2 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //获取cookie
        Cookie[] cookies = request.getCookies();
        //遍历数组
        for (Cookie cookie : cookies) {
            //获取cookie中的key
            String name = cookie.getName();
            //获取cookie中的value
            String value = cookie.getValue();

            System.out.println(name+"..."+value);
        }
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 3.Cookie的时效性

1.时效性:其实就是指的是Cookie的一个存活时间
2.默认情况下:Cookie只存在一次会话中

# 3.1.会话级Cookie

1.服务器不指定Cookie的存活时间的情况下,Cookie只存活在一次会话中
2.浏览器打开连接到服务器之后,服务器创建Cookie对象,保存到浏览器中
3.只要是在一次会话中(浏览器和服务器不断开),都可以拿到Cookie中的数据

# 3.2.持久化Cookie

1.在服务器端创建cookie的时候指定存活时间,此时cookie被设置的时间控制,会存在硬盘上,超时,cookie就没有了
2.只要是在这个时间里面,都可以拿到Cookie,超过时间,cookie消失

# 3.3.设置Cookie存在时间方法

方法说明
cookie.setMaxAge(cookie存活时间)时间单位为:秒
@WebServlet("/cookie1")
public class Cookie1 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //1.创建Cookie对象
        Cookie cookie = new Cookie("username", "aobama");

        //设置cookie存活时间
        cookie.setMaxAge(20);

        //2.将cookie响应给浏览器
        response.addCookie(cookie);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}


@WebServlet("/cookie2")
public class Cookie2 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //获取cookie
        Cookie[] cookies = request.getCookies();
        //遍历数组
        for (Cookie cookie : cookies) {
            //获取cookie中的key
            String name = cookie.getName();
            //获取cookie中的value
            String value = cookie.getValue();

            System.out.println(name+"..."+value);
        }
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

1.在cookie1中设置了cookie的存活时间,为20秒,当执行cookie1时,cookie数据已经就保存到浏览器上了

2.访问cookie2能从cookie中将数据获取出来

3.如果关闭浏览器,在20之内打开,重新访问cookie2,还是能够将cookie获取,超时之后,拿不到了

# 4.Cookie设置domain和path(了解说明即可)

1.上网时间长了，本地会保存很多Cookie。对浏览器来说，访问互联网资源时不能每次都把所有Cookie带上。

2.比如说,我们访问过京东,百度,那么浏览器上就存了京东相关的cookie和百度相关cookie
  后面如果要是在访问京东,我们浏览器就不能把百度的cookie数据代入到京东,反过来也是一样的不能

3.解决:
  我们可以通过调用cookie的setPath()和setDomain()方法来设置cookie的domain和path
  浏览器会使用Cookie的domain(网站域名)和path属性值(访问的部署的web应用上下文名称路径)来和当前访问的地址进行比较，从而决定是否携带这个Cookie。

# 第三章.服务器会话技术_Session(重中之重)

# 1.Session介绍

1.概述:
  session是服务器端的一个技术,可以往里面存放数据,只要是在一次会话中的多个请求之间,都可以获取session中的数据,会话结束,session中的原始数据获取不到了
      
2.作用:
  作为域对象存储数据在服务器端
      
3.注意:服务器会为每一个浏览器在内存中开辟一个空间,就是session

# 2.Session入门

方法说明
request.getSession()获取HttpSession对象
void setAttribute(String name,Object value)存数据
Object getAttribute(String name)取数据
void removeAttribute(String name)删除数据
  @WebServlet("/session1")
public class Session1 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        HttpSession session = request.getSession();
        session.setAttribute("username","aobama");
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

@WebServlet("/session2")
public class Session2 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        HttpSession session = request.getSession();
        Object username = session.getAttribute("username");
        System.out.println(username);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 3.Session工作原理和有效期

# 3.1.Session工作原理

当我们创建Session,往里面存数据的时候,服务器,会自动创建一个Cookie对象(tomcat创建),往Cookie中设置一个JSESSIONID

如果从session中获取的时候,Cookie中的JSESSIONID和服务器上的JSESSIONID一样,就能成功获取session中的数据


当会话结束(浏览器关闭),再想获取session中的数据,发现,获取不到了
因为当会话结束了,原有的JSESSIONID丢失了,和之前的不一样了,所以获取不到

session中的数据,是由保存到Cookie中的JSESSIONID去维护的

只要是会话不结束,或者在Cookie设置的时间范围内,都可以根据同样的JSESSIONID去找到session对象,获取session对象中的数据

# 3.2.设置Session有效期_了解即可

方法说明
session.setMaxInactiveInterval(秒)设置session闲置超时时间
session.invalidate()强制Session立即失效

1.session对象默认存活时间是30分钟

2.关闭浏览器,不是session对象销毁了,而是cookie没了,倒是JESSSIONID对不上了

# 2.将登录的用户放到session中

当我们登录之后,那么在页面上怎么点都应该显示登录的用户名,所以我们需要将登录的用户名放到session中

# 2.1.Servlet层_UserServlet

//用户登录
    public void doLogin(HttpServletRequest request, HttpServletResponse response) throws IOException {
        //获取页面提交的参数
        String username = request.getParameter("username");
        String password = request.getParameter("password");
        String pwd = MD5Utils.encode(password);
        try {
            //调用Service层方法
            User user = service.loginUser(username, pwd);
            if (user != null) {
                //===================以下是修改的后台代码=========================
                /*
                  用户成功登录之后,先不要转发到页面,先把登录成功的用户放到session域中
                  只要浏览器不关,在每个页面都能拿到session中的数据
                 */
                HttpSession session = request.getSession();
                session.setAttribute("user",user);
                //===================以上是修改的后台代码=========================
                
                //转发到登录成功页面
                processTemplate("user/login_success", request, response);
            } else {
                //将错误提示放到域中
                request.setAttribute("loginMessage", "对不起,登录失败!");
                //将页面重新跳转到登录页面
                processTemplate("user/login", request, response);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

public void logout (HttpServletRequest request, HttpServletResponse response) {
        //销毁session
        request.getSession().invalidate();
        /*
          销毁之后还要继续在首页中

          首页是通过PortalServlet设置的url-pattern进去的

          所以我们直接可以重定向到PortalServlet上,再次通过
          PortalServlet设置的url-pattern重新进入首页

         */

        try {
            response.sendRedirect(request.getContextPath()+"/index.html");
        } catch (IOException e) {
            e.printStackTrace();
        }

    }

# 4.第三方验证码_kaptcha

# 4.1.验证码介绍

通过让用户填写验证码并在服务器端检查，防止浏览器端使用程序恶意访问。

我们点验证码是请求,写完验证码点击"注册"也是请求,所以用session合适

# 4.2.第三方验证码_kaptcha

1.导入kaptcha-2.3.2.jar
2.jar包中已经写好了Servlet的Java类，我们只需要在web.xml中配置这个Servlet即可。

<!--
   kaptcha 验证码Servlet配置
-->

    <servlet>
        <servlet-name>kaptchaServlet</servlet-name>
        <servlet-class>com.google.code.kaptcha.servlet.KaptchaServlet</servlet-class>

        <!-- 通过配置初始化参数影响KaptchaServlet的工作方式 -->
        <!-- 可以使用的配置项参考com.google.code.kaptcha.util.Config类 -->
        <!-- 配置kaptcha.border的值为no取消图片边框 -->
        <init-param>
            <param-name>kaptcha.border</param-name>
            <param-value>no</param-value>
        </init-param>
    </servlet>
    <servlet-mapping>
        <servlet-name>kaptchaServlet</servlet-name>
        <url-pattern>/kaptcha</url-pattern>
    </servlet-mapping>

 /**
     * 重构注册功能
     * doRegister
     */

    public void doRegister(HttpServletRequest request, HttpServletResponse response){
        try{
            Map<String, String[]> map = request.getParameterMap();
            User user = new User();
            BeanUtils.populate(user,map);
            user.setPassword(MD5Utils.encode(user.getPassword()));

            String kaptcha_session_key = (String)request.getSession().getAttribute("KAPTCHA_SESSION_KEY");

            String code = map.get("code")[0];
            if (code.equalsIgnoreCase(kaptcha_session_key)){
                int result = userService.registerUser(user);

                if (result==1){
                    processTemplate("user/regist_success.html",request,response);
                }else{
                    request.setAttribute("message","注册失败");
                    processTemplate("user/regist.html",request,response);
                }
            }else{
                request.setAttribute("messageCode","验证码不对");
                processTemplate("user/regist.html",request,response);
            }

        }catch (Exception e){
            e.printStackTrace();
        }

    }
