# 第一章.Web资源

# 1.Web资源的介绍

1.web资源就是运行在服务器上的资源
  比如:web目录下的js  页面 文件 图片 css等
      
2.web资源分为两大类
  a.静态web资源
  b.动态web资源

# 2.静态Web资源

静态资源就是web页面中供人们浏览的数据,这些数据始终是不变的,比如:html  css  js  图片等都属于静态web资源
    
不需要查询数据库,也不需要程序处理,直接就能显示在页面上,如果想修改内容则必须修改页面

# 3.动态Web资源

动态web资源就是web页面中供人们浏览的数据是由程序产生的,不同的用户或者不同的时间点访问web页面看到的内容不一样,例如:Servlet jsp  Thymeleaf等都是动态web资源
    
需要程序处理或者从数据库中读数据,能根据不同的条件在页面上显示不同的效果,内容更新不需要修改页面

# 第二章.Servlet

# 1.Servlet介绍

1.概述:
  Servlet是运行在服务器端的一个小程序,是sun公司制定的一个动态web资源标准,从代码层面上来看Servlet是一个接口
2.作用:
  a.接收客户端发到服务端的请求,分析请求
  b.可以将结果响应给客户端

# 2.Servlet入门

需求:
  访问一个servlet,响应一个"helloservlet"

1.创建一个类,实现Servlet接口(tomcat提供好了这个接口)
2.重写Servlet接口中的方法
  主要在service方法中做接收请求,回响应操作
3.配置servlet信息
4.在浏览器上访问servlet

# 6.Servlet生命周期

# 6.1.Servlet生命周期介绍

1.概述:当我们将servlet部署到tomcat下,我们从启动tomcat,到访问servlet,再到关闭tomcat,在这个过程中,不同的操作会触发不同的生命周期
2.servlet对象什么时候被创建:第一次访问servlet的时候,tomcat会创建servlet对象,只创建一次
  servlet对象生命时候被销毁:关闭tomcat,或者将项目从tomcat下移除
      
3.每到一个生命周期节点,都会调用对应的生命周期方法:
  init()->第一次访问servlet的时候
  service()->每次发请求会有执行一次
  destroy()->关闭tomcat,或者将项目从tomcat下移除
      
4.三个方法有啥用?
  init()一般可以将需要最先初始化的数据,写到init中
  service()接收请求,处理响应
  destroy()释放资源

@WebServlet("/servlet4")
public class Servlet4 implements Servlet {
    @Override
    public void init(ServletConfig servletConfig) throws ServletException {
        System.out.println("我出生了!");
    }

    @Override
    public ServletConfig getServletConfig() {
        return null;
    }

    @Override
    public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
        System.out.println("我接收到请求了");
    }

    @Override
    public String getServletInfo() {
        return null;
    }

    @Override
    public void destroy() {
        System.out.println("我死了");
    }
}

# 6.2. Servlet对象提前创建

1.问题描述:
  如果用户第一次访问的时候,servlet对象才能创建,用户会觉得比较慢,这样不好
2.解决:
  等到项目一启动servlet就马上创建,服务器可能会觉得比较慢,但是用户端会比较快,所以我们需要牺牲一下,为了用户体验
      
3. <load-on-startup>数字</load-on-startup>  这个配置在web.xml中的<servlet>标签中写
   数字越小,servlet越先被创建

    <servlet>
        <servlet-name>servlet5</servlet-name>
        <servlet-class>com.atguigu.a_servlet.Servlet5</servlet-class>
        <load-on-startup>2</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>servlet5</servlet-name>
        <url-pattern>/servlet5</url-pattern>
    </servlet-mapping>

    <servlet>
        <servlet-name>servlet6</servlet-name>
        <servlet-class>com.atguigu.a_servlet.Servlet6</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>servlet6</servlet-name>
        <url-pattern>/servlet6</url-pattern>
    </servlet-mapping>

public class Servlet7 implements Servlet {

# 8.1.描述

3.GenericServlet下有一个子类HttpServlet

# 8.2.继承HttpServlet方式实现Servlet

@WebServlet("/httpservlet1")
public class Servlet1 extends HttpServlet {
    /**
     * 如果发送get请求,那么doGet方法就要执行
     * @param req
     * @param resp
     * @throws ServletException
     * @throws IOException
     */
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.getWriter().write("i am httpservlet1");
    }

    /**
     * 如果发送post请求,那么doPost方法就要执行
     * @param req
     * @param resp
     * @throws ServletException
     * @throws IOException
     */
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
       doGet(req,resp);
    }
}

# 8.3.doGet()和doPost方法介绍

1.doGet():用来处理前端发过来的get请求的(method = "get")
  a标签超链接和地址栏上的请求都是get请求
    
2.doPost():用来处理前端发过来的post请求的(method = "post")
    
3.注意:
  我们将来不可能针对doGet写一遍功能,针对doPost方法再写一遍同样的功能,所以针对于服务器来说,get或者post其实都一样
  我们可以在doPost方法中直接调用doGet方法;反过来也行

@WebServlet("/httpservlet1")
public class Servlet1 extends HttpServlet {
    /**
     * 如果发送get请求,那么doGet方法就要执行
     * @param req
     * @param resp
     * @throws ServletException
     * @throws IOException
     */
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.getWriter().write("i am httpservlet1");
    }

    /**
     * 如果发送post请求,那么doPost方法就要执行
     * @param req
     * @param resp
     * @throws ServletException
     * @throws IOException
     */

# 2.ServletContext对象获取web.xml中全局初始化参数

方法名说明
String getInitParameter(String key)
key指的是<param-name>中的内容		根据设置的<param-name>获取<param-value>
String getRealPath("web资源的相对路径")获取web应用程序下,任意资源的绝对路径(文件下载会用到)
String getContextPath()获取web应用程序的名字(部署项目时application context中设置的web名字)
String getMimeType("xxx.文件后缀名")获取文件的MIME类型(此MIME是给浏览器看的)
浏览器看不懂.doc .jpg这种扩展名
所以tomcat在conf路径下的web.xml中规定了每一个扩展名对应的MIME类型
@WebServlet("/myservlet5")
public class MyServlet5 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        ServletContext servletContext = getServletContext();
        /*
          String getInitParameter(String key) key指的是<param-name>中的内容
         */

        String context1 = servletContext.getInitParameter("context1");
        System.out.println("context1 = " + context1);

        System.out.println("================================");

        /*
          String getRealPath("web资源的相对路径")获取web应用程序下,任意资源的绝对路径(文件下载会用到)

          问题:为什么要根据相对路径获取绝对路径
          原因:我们开发好的web程序给用户,用户需要将打包好的web程序放到自己电脑的tomcat下去运行
              用户将web资源放到哪里我们其实是不知道的,所以我们可以通过相对路径去获取用户所放web资源的绝对路径
         */

        /*
          获取一下web路径下的1.txt

          在webapp下是没有web目录的,所以我们直接用1.txt就能获取1.txt在硬盘上的绝对路径
         */
        String realPath1 = servletContext.getRealPath("1.txt");
        System.out.println("realPath1 = " + realPath1);

        /*
          获取WEB-INF下的2.txt
         */
        String realPath2 = servletContext.getRealPath("WEB-INF/2.txt");
        System.out.println("realPath2 = " + realPath2);

        /*
          获取src下的3.txt

         */
        String realPath3 = servletContext.getRealPath("WEB-INF/classes/3.txt");
        System.out.println("realPath3 = " + realPath3);

        System.out.println("=================================");

        /*
          String getContextPath()-> 获取的是部署时在application context中写的web项目名
         */

        String contextPath = servletContext.getContextPath();
        System.out.println("contextPath = " + contextPath);

        System.out.println("====================================");

        /*
          String getMimeType("xxx.文件后缀名")->根据指定类型的文件获取对应的MIME类型
         */

        String mimeType = servletContext.getMimeType("a.jpg");
        System.out.println("mimeType = " + mimeType);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

# 3.ServletContext 域对象

1.概述:域对象可以看成是一个容器,这里面存数据
      数据的格式都是键值对形式
    
2.作用:在ServletContext域中存储的数据,在当前项目下的任何一个servlet中都可以获取到
    
3.注意:ServletContext是域对象中作用范围最大的一个
方法名说明
void setAttribute(String key,Object o)存数据,key是String;value类型任意
Object getAttribute(String key)获取数据,根据key获取value
void removeAttribute(String key)删除数据
@WebServlet("/myservlet6")
public class MyServlet6 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        ServletContext servletContext = getServletContext();
        //存数据
        servletContext.setAttribute("username","root");

        //删数据
        servletContext.removeAttribute("username");
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

@WebServlet("/myservlet7")
public class MyServlet7 extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        ServletContext servletContext = getServletContext();
        //获取ServletContext域中的数据
        Object username = servletContext.getAttribute("username");
        System.out.println("username = " + username);
    }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}
